## PR Review Checklist

When reviewing code changes:

1. **Code Quality** — Is the code clean and readable?
2. **Security** — Are there any security vulnerabilities?
3. **Performance** — Could anything cause slowdowns?
4. **Best Practices** — Does it follow React/TypeScript standards?
5. **Tests** — Are there tests for new functionality?

## Format
- Start with a brief summary
- List issues by severity (Critical, Warning, Suggestion)
- End with overall recommendation (Approve/Request Changes)
